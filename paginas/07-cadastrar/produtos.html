7.4 Catálogo de Produtos e Embalagens

O cadastro de produtos é a base para toda a inteligência do WMS. Aqui definimos como o item se comporta fisicamente, quais seus códigos de barras e quais as regras de saída.

Master-Detail Catalog

Configurações Essenciais

  • Código do Produto: SKU único (ex: VEPEL-BPI-174FX).
  • Tipo / Família: Define a categoria técnica (ex: Motor, Cabine, Freio).
  • Movimenta Estoque: Ative para peças físicas. Desative para serviços ou itens virtuais.
  • Nota de Negócio: A VerticalParts não utiliza data de validade. O controle de giro é puramente logístico/qualitativo.

Um produto pode ter múltiplas formas de ser bipado (unidade, caixa, palete).

  • Cód. de Barras (EAN): O bipador mobile reconhecerá o item por este código.
  • Fator de Conversão: Ex: se uma caixa tem 12 unidades, o fator é 12.
  • Lastro e Camadas: Define o tamanho padrão do palete para cálculos de ocupação volumétrica.
  • Cálculo do Palete: O sistema calcula automaticamente o peso e a quantidade total por palete completo.
  1. FIFO (First In, First Out): Recomendado para a maioria das peças. Expede o lote que entrou primeiro.
  2. LIFO (Last In, First Out): Raramente usado, exceto em fluxos específicos de contrapeso.
  3. LOC (Sequência de Local): Otimiza a rota do separador, priorizando o item mais próximo da saída ou da doca.
💡 Dica: Você pode usar a função "Gerar EAN-13" para criar códigos internos caso o fabricante não forneça etiqueta de barras.
✓ Integração Omie: O catálogo é sincronizado automaticamente. Se você cadastrar um produto no Omie, ele aparecerá aqui para ser "enriquecido" com os dados logísticos (embalagem/endereço).
